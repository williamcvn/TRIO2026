# TRIO2026 DevLauncher — 機台螢幕模擬器使用說明

> **撰寫**: Office of William  
> **日期**: 2026-04-29  
> **版本**: 2.0

---

## 概述

**DevLauncher** 是一個 GUI 容器模擬器，模擬機台觸控螢幕的實際顯示效果。它提供一個可調整大小的視窗框架，選擇目標解析度後，視窗會自動等比例縮放至開發機可顯示的大小，點擊啟動後會將 TRIO2026.App 嵌入其中滿版呈現。

> [!NOTE]
> 此工具僅供開發與測試使用，不會部署至正式機台。  
> 平時開發仍可在 VS 中直接啟動 TRIO2026.App（全螢幕模式）。

---

## 操作流程

```
┌───────────────────────────────────────┐
│  [🖥️ 螢幕模擬: 1920×1200 ▼]  [▶ 啟動] [■ 停止]  │  ← 工具列
├───────────────────────────────────────┤
│                                       │
│          TRIO2026.App 嵌入區域         │  ← 模擬螢幕
│        （等比例縮放後滿版呈現）         │
│                                       │
├───────────────────────────────────────┤
│  模擬: 1920×1200 | 縮放: 78%          │  ← 狀態列
└───────────────────────────────────────┘
```

### 步驟

1. **啟動 DevLauncher**（VS 設為啟動專案或命令列 `dotnet run --project tools\DevLauncher`）
2. **選擇解析度**：從下拉選單選擇目標螢幕尺寸
3. **觀察縮放**：視窗自動調整大小，狀態列顯示實際縮放比例
4. **點擊「▶ 啟動」**：TRIO2026.App 嵌入模擬器中滿版呈現
5. **點擊「■ 停止」**：結束 App 進程

## 等比例縮放機制

模擬器會自動計算開發機螢幕可容納的最大顯示尺寸：

- 若開發機解析度 ≥ 模擬解析度 → **1:1 顯示**（縮放 100%）
- 若開發機解析度 < 模擬解析度 → **等比例縮小**至開發機可容納的範圍

例：開發機 1920×1080，模擬 1920×1200：
- 縮放比例 ≈ 78%（受高度限制）
- 實際顯示尺寸 ≈ 1497×936

## 技術原理

### 嵌入機制

使用 Win32 `SetParent` API 將 TRIO2026.App 的視窗嵌入 DevLauncher 的容器區域：

1. 啟動 `TRIO2026.App.exe` 並傳入 `--sim-embedded 1` 參數
2. App 收到此參數後：不啟用透明背景、不全螢幕
3. DevLauncher 等待 App 視窗出現
4. 使用 `SetParent()` 設定父視窗
5. 使用 `MoveWindow()` 調整子視窗填滿容器

### 命令列參數

```
TRIO2026.App.exe --sim-width 1920 --sim-height 1200 --sim-touch 1 --sim-fullscreen 0 --sim-embedded 1
```

| 參數 | 說明 |
|------|------|
| `--sim-width` | 視窗寬度 |
| `--sim-height` | 視窗高度 |
| `--sim-touch` | 模擬觸控 (0/1) |
| `--sim-fullscreen` | 全螢幕模式 (0/1) |
| `--sim-embedded` | 嵌入模式 (0/1) |

## TRIO2026.App 啟動模式比較

| 啟動方式 | 模式 | 透明背景 | 視窗大小 |
|---------|------|---------|---------|
| VS 直接啟動（無參數） | 正常模式 | ✅ | 全螢幕最大化 |

## 注意事項

> [!IMPORTANT]
> 使用 DevLauncher 前必須先建置 TRIO2026.App：
> ```powershell
> dotnet build src\TRIO2026.App
> ```

> [!WARNING]
> 嵌入模式下 `AllowsTransparency` 會被關閉（與 Win32 SetParent 不相容），因此登入頁背景不透明。這不影響正式機台的全螢幕體驗。

> [!TIP]
> 切換解析度時不需要重新啟動 — 直接從下拉選單選擇即可。但如果 App 已在運行中，需先停止再重啟。
